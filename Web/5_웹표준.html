웹표준

웹 사이트를 만들 때 표준적인 규칙

W3C(www 컨소시엄)가 정하고 이에 대한 장려를 추진한다.

과거 각각의 브라우저가 경쟁을 목적으로 다른 브라우저와 호환이 되지않는 요소를 사용.
그로인해 브라우저 별로 보이는 화면이 다르고 개발도 각각 해야함

어떤 환경에서도 이용할 수 있는 웹 페이지를 만들기 위함

표준기술은 - (x)HTML, CSS, XML, DOM, ECMAScript

각 기술들을 표준화된 방법으로 사용해야 한다.

HTML에 DOCTYPE 선언
시멘틱 태그 사용
css를 사용하여 내용과 디자인 분리
등등

유지보수 용이, 웹 접근성 향상, 검색엔진 최적화, 브라우저간 호환 가능 등 장점이 있다.

웹 접근성

장애인, 고령자 등 누구나 웹 페이지를 이용할 수 있어야한다.

웹 접근성을 위해 사용되는 보조기기는 자막, 스크린리더, 자동완성 기능, 마우스스틱 등이 있다.

웹 표준을 지키는건 웹 접근성을 향상시키는 가장 기본적인 방법이다.

웹 호환성 - 크로스 브라우징

웹 페이지 제작 시 모든 브라우저에서 의도한 대로 보여지고 작동되는 기법

브라우저 별로 렌더링 엔진이 다름
작동하지 않는 HTML, JS 코드 존재
해석하지 못하는 CSS 코드 존재
브라우저 자체적인 CSS 스타일

대표적인 크로스 브라우징 작업 방법

CSS 초기화

모질라 MDN (https://developer.mozilla.org/ko/) 호환성 확인

Can I Use.. (https://caniuse.com/?search=date) 호환성 확인

CSS 벤더 프리픽스(-webkit-, -ms-) 사용
