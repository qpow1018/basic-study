REST API

API란

Application Programming Interface

Interface - 상호간에 소통을 위해 만들어진 물리적 매개체나 방법

프로그램간 데이터를 주고 받기 위한 방법

URI과 URL

URL - Uniform Resource Locator - 자원이 실제로 존재하는 위치, 즉 주소

URI - Uniform Resource Identifier - 자원의 고유 식별자로서 URL의 의미를 포함한다

REST - Representational State Tranfer

자원을 이름으로 구분하여 해당 자원의 상태를 주고 받는 모든 것

REST는 HTTP을 그대로 활용한다

HTTP URI를 통해 자원을 명시하고 HTTP Method를 통해 해당 자원에 대한 CRUD를 적용하는 것

구조화된 데이터 표현이 가능하고 가벼운 JSON이 많이 사용된다.

웹의 장점을 최대한 활용할 수 있는 아키텍처 스타일(구조라고 이해하자)

REST 형식의 API

각 요청이 어떤 정보나 동작을 위한 것인지 그 모습 자체만으로 추론이 가능하다

동사는 URI에 표현하지 않는다 - Method로 표현
