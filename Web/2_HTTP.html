프로토콜이란

사람과 사람이 대화(통신)할때 서로 이해할 수 있는 언어, 공용된 언어를 사용해야 전 세계 사람과 대화할 수 있다.

컴퓨터와 컴퓨터도 통신(대화)할때 서로 이해할 수 있는 언어, 공용된 언어를 사용해야 통신할 수 있다.

프로토콜은 컴퓨터간 통신할때 공용 언어를 사용하자는 규칙, 규약

즉, 컴퓨터간 원활한 통신을 위해 약속된 규칙

내가 들어본건 HTTP, FTP 정도. 그 외에도 많지만 우선 HTTP에 집중하자

HTTP

HyperText Transfer Protocol

웹에서 데이터를 주고받을 수 있는 프로토콜

요청과 응답 형태

클라이언트 - 서버 구조

무상태성 (statekess) - 서버가 클라이언트의 상태를 저장하지 않는다.
요청 시 마다 필요한 정보를 다 보내야 한다.
아무 서버나 호출해도 된다. - 확장성 좋음, 특정 서버를 기다리지 않아도 된다.

비연결성 (connectionless) - 연결을 지속하지 않는다.
클라이언트가 요청 후 응답을 받으면 연결을 끊어버린다 - 서버의 자원을 필요할때만 사용한다.
다만, 요청 시 마다 계속 연결을 해야하기 때문에 추가적인 css, js 등을 받아야 하기에 비효율적이다.
'HTTP 지속 연결'로 해결 - 연결이 이뤄지고 모든 자원을 받을때까지 기다렸다가 연결을 끊는다.
